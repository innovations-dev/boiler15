## Quick Start

1. Clone the repository:

```bash
git clone https://github.com/innovations-dev/boiler15.git my-app
cd my-app
```

2. Install dependencies:

```bash
pnpm install
```

## Environment Setup

1. Copy the environment variables template:

```bash
cp .env.sample .env
```

2. Configure your environment variables:

```bash
App
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_GOOGLE_SITE_VERIFICATION=your_verification_code
Auth
BETTER_AUTH_URL=your_auth_url
BETTER_AUTH_SECRET=your_secret
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret
Email
RESEND_API_KEY=your_resend_api_key
EMAIL_FROM=your_email@domain.com
TEST_EMAIL=test@example.com
Database
TURSO_DATABASE_URL=your_turso_url
TURSO_AUTH_TOKEN=your_turso_token
```

> **Note:** We use [@t3-oss/env](https://github.com/t3-oss/env) for type-safe environment variables.

## Third-Party Services Setup

### 1. Database Setup

We use [Turso](https://turso.tech/) for our database. To set it up:

1. Create a Turso account
2. Install the Turso CLI:
   ```bash
   curl -sSfL https://get.tur.so/install.sh | bash
   ```
3. Create a new database:
   ```bash
   turso db create my-app
   ```
4. Get your credentials:
   ```bash
   turso db tokens create my-app
   ```
5. Update your `.env` file with the database URL and token

### 2. Email Service

1. Create a [Resend](https://resend.com) account
2. Get your API key from the dashboard
3. Add it to your `.env` file as `RESEND_API_KEY`

### 3. Authentication

1. Configure your OAuth providers (if using social login):
   - GitHub: Create OAuth app at GitHub Developer Settings
   - Google: Set up OAuth 2.0 credentials in Google Cloud Console
2. Add the client IDs and secrets to your `.env` file

## Database Migration

Initialize and migrate your database:

```bash
pnpm db:generate
pnpm db:migrate
```

## Development Server

Start the development server:

```bash
pnpm dev
```

Your application should now be running at `http://localhost:3000`
